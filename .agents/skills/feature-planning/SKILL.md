---
name: feature-planning
description: Turn a SaaS feature idea into requirements, acceptance criteria, risks, and implementation slices.
---

## Required context

Before acting:

1. Read `AGENTS.md`.
2. Read `docs/ai-project-memory/00-project-index.md`.
3. Read the current branch memory file if one exists.
4. Inspect actual repository files before making conclusions.

## Workflow

1. Have `product_manager` define user, value, plan impact, acceptance criteria, and non-goals.
2. Have `staff_architect` define affected systems and safest implementation slice.
3. Identify auth, billing, tenant, data, and deployment implications.
4. Update branch memory with the planned scope.

## Output

Return:

- user story
- acceptance criteria
- implementation slices
- affected systems
- test requirements
- security/billing considerations
